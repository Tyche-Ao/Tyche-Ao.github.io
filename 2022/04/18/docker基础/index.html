服务

  1. 启动服务

    systemctl start docker

  2. 停止服务

    systemctl stop docker

  3. 重启服务

    systemctl restart docker

  4. 查看服务状态

    systemctl status docker

  5. 设置开机启动服务

    systemctl enable docker

镜像

  1. 查看镜像

    docker images

    docker images -q 查看所有镜像的id

  2. 搜索镜像

    docker search 镜像名称

  3. 拉取镜像

    docker pull 名称:版本

  4. 删除镜像

    docker rmi 镜像id

    docker rmi 版本名称:版本

    docker rmi docker images -q

容器

  1. 查看容器

    docker ps 查看正在运行的容器

    docker ps -a

  2. 创建

    docker run -it –name 容器名 centos:7（镜像名） /bin/bash

    -i 保持容器运行。

    -t 交互式容器。 与-i一起使用。创建后自动进入容器，退出后容器关闭

    -d 守护式容器。创建一个容器在后台运行。使用docker exec进入容器。退出后容器不会关闭。

  3. 进入

    docker exec -it 容器名 /bin/bash

  4. 启动

    docker start 容器名

  5. 停止

    docker stop 容器名

  6. 删除

    docker rm 容器名/id

  7. 查看容器信息

    docker inspect 容器名

  8. 容器内退出

    exit Ctrl+D

数据卷

宿主机当中的一个目录/文件

数据卷的作用

  • 容器数据的持久化
  • 外部机械和容器间的通信
  • 容器之间的数据交换

数据卷的创建

docker run -it –name 容器名 -v ~/data:/root/data_container centos:7 /bin/bash

在容器创建时用-v创建数据卷。~等于/root， 宿主机目录:容器目录 ，都必须用绝对路径，目录不存在会自动创建。

数据卷容器

创建数据卷容器

docker run -it –name c3 -v /volume centos:7

将其他容器 绑定到数据卷容器上

docker run -it –name a1 –volumes-from c3 centos:7

端口映射

部署Mysql

docker run -id \

-p 3306:3306 \

-v $PWD/conf:/etc/mysql/conf.d \

-v $PWD/logs:/logs \

-v $PWD/data:/var/lib/mysql \

-e MYSQL_ROOT_PASSWORD=126 \

mysql:8.0

  • -p 表示端口映射，将容器端口映射到宿主机端口，宿主机端口：容器端口
  • $PWD 表示当前目录
